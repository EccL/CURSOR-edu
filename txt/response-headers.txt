Response headers : 

cache-control - правила кешування ресурсу. Директива no-cache означає, що при повторному запиті до цього ж URL відповідь можна використати лише після перевірки змін;

content-encoding - спосіб кодування вмісту при передачі. Вказує, який алгоритм був використаний для даного повідомлення;

Content-Security-Policy - дозволяє знизити ризики XSS на сучасних браузерах, оголосивши, які динамічні ресурси дозволені для завантаження через HTTP Header;

content-type - тип повернутого документа; опис даних, що містяться в body;

date - дата генерування запиту;

expire - повідомляє усім кешам як довго контент трактується як свіжий; по закінченню цього часу кеш завжди буде запитувати сервер чи не було змін контенту;

Last-Modified - дата останньої модифікації контенту;

p3p - протокол, що дозволяє сайтам інформувати браузер про можливе отримання персональних даних користувача;

pragma - реалізація конкретних директив, які можуть застосовуватися на протязі усієї request/response сесії.

Public Key Pinning - прив'язка публічного ключа до мережевого ресурсу (по імені чи адресі) на стороні клієнта. Дозволяє виявити підміну ключів;

server - список імен і версій веб-сервера і його компонентів. Для проксі-серверів поле - Via;

set-cookie - відправляється лише тоді, коли сервер бажає щоб браузер зберіг кукі;

status - статус запиту;

transfer-encoding - список способів кодування, які були застосовані до повідомлення для передачі;

vary - список, який описує ресурс з полів запиту, які були взяті до уваги;

Strict Transport Security - механізм, що активує форсоване захищене зєднання через протокол HTTPS. Дозволяє зразу ж встановлювати безпечне з'єднання, замість використання HTTP-протоколу;

x-content-type-options - вказує, що немає необхідності автоматичного визначення Content-Type, а треба використовувати вже повернутий content-type; 

X-XSS-Protection - підвищення безпеки вашого сайту стосовно деяких видів XSS (cross-site scripting) атак;

x-frame-options - дозволяє або забороняє завантаження сторінки сайту в <iframe></iframe>, який знаходиться на іншому домені."Показує" браузеру, що треба робити, якщо сайт завантажено у <frame>;

x-sdch-encode - SDCH encoder/decoder для node.js (?!); 

x-amz-id-... - заголовки які amazon додає до усіх відповідей;

x-fb-debug - ідентифікатор відсліджування запиту (Facebook).
