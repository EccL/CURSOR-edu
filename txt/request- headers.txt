Request headers :
 
HTTP метод / HTTP версія;

accept - список допустимих форматів ресурсу; 

accept-encoding - перелік способів кодування, які підтримуються при передачі;

accept-language - перелік мов, що підтримуються;

cache-control - основні дерективи для керування кешуванням;

connection - дані про проведення з'єднання;

cookie - HTTP кукі, які раніше відправив сервер;

host - доменне ім'я та порт хоста ресурсу до якого зверталися. Необхідно для підтримки віртуального хостингу на серверах;

referer - URL ресурсу, після якого клієнт зробив поточний запит;
 
upgrade-insecure-requests - дозволяє автоматично звертатись до сайту по https://, якщо він працює в двох режимах (http и https) і 
забезпечує цілісність даних при передачі їх від користувача до сайту і назад;

user-agent - список назв та версій клієнта та його компонентів (з коментарями); 

x-client-data - не містить ніяких ідентифікаційних даних, а тільки інформацію про встановлений екземпляр Chrome (включно з активними модифікаціями).
